# Security Policy

LoginGuard is a defensive, educational, authorized security assessment tool. The project welcomes responsible reports about vulnerabilities in LoginGuard itself.

## Supported Versions

This project is in early development. Security fixes are handled on the main development line until formal releases are established.

## Reporting a Vulnerability

Please do not disclose suspected vulnerabilities publicly until maintainers have had a reasonable opportunity to review and address them.

To report a vulnerability:

1. Open a private security advisory if the hosting platform supports it.
2. If private advisories are not available, contact the maintainers through the project repository and request a private reporting channel.
3. Include a clear description, affected files or versions, reproduction steps, and potential impact.

Do not include real credentials, private user data, or data from systems you are not authorized to test.

## Ethical Boundaries

Reports must focus on LoginGuard and systems you own or have explicit permission to assess. The project does not support unauthorized testing, offensive payloads, credential collection, brute force workflows, or instructions for attacking third-party systems.

## Maintainer Response

Maintainers will aim to:

- Acknowledge valid reports promptly.
- Ask clarifying questions when needed.
- Prioritize fixes based on impact.
- Credit reporters when appropriate and requested.
- Publish a security note or changelog entry after a fix when useful.
