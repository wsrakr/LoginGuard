# v0.2.0 - Lab Check System and Firefox Groundwork

Release status: Ready for release

LoginGuard v0.2.0 improves Lab Mode into a clearer controlled-check workflow for local and authorized lab users, while adding browser API groundwork for future Firefox/WebExtensions compatibility.

This release does not change the core safety model. Website Check remains passive and local-first. Lab Mode remains restricted to local or explicitly authorized lab contexts.

## Summary

v0.2.0 focuses on product clarity, Lab Mode structure, and future browser compatibility preparation.

The release introduces clearer Lab Mode check definitions, planner-only workflows for future checks, a polished persistent Lab Session workspace, and a small browser API adapter that reduces direct coupling to Chrome-specific extension APIs. Chrome/Chromium remains the current primary supported environment. Firefox compatibility is future groundwork only and should not be described as supported until testing is completed and documented.

## Highlights

| Area | What Changed |
| --- | --- |
| Lab Check Registry | Centralized user-facing labels, descriptions, availability states, safety levels, and technical category IDs for Lab Mode checks. |
| Empty Fields Observation Planner | Added a planner-only check for future required-field behavior review in local or authorized lab contexts. |
| Response Message Comparison Planner | Added a planner-only check for future visible response-message comparison in controlled labs. |
| Persistent Lab Session Polish | Improved the persistent Lab Session page with clearer summary, check cards, safety boundaries, actions, and technical details. |
| Browser API Adapter Groundwork | Added a small compatibility layer for current extension API usage to prepare for future WebExtensions work. |
| Website Check | Continued passive current-page Website Check behavior without adding active testing. |

## Lab Mode Safety Model

Lab Mode remains controlled and restricted.

- Website Check remains passive.
- Lab Mode remains local or authorized only.
- Planner-only checks do not submit forms.
- LoginGuard does not collect credentials.
- LoginGuard does not read passwords or input values.
- LoginGuard does not run payloads.
- LoginGuard does not perform brute force or password spraying.
- LoginGuard does not perform bypass automation.
- Invalid Credentials Check remains blocked for now.

Baseline Observation Executor v0 remains metadata-only. It records approved page/form metadata in allowed local lab contexts after the existing safety gates and confirmation flow. It does not submit forms, trigger clicks, change values, read input values, collect response bodies, or collect credentials.

## Browser Support

Chrome/Chromium remains the primary supported environment for the current prototype.

Firefox/WebExtensions support is not claimed in v0.2.0. This release adds groundwork only:

- Browser support planning documentation.
- A small browser API adapter for low-risk extension API calls.
- Compatibility principles for future Firefox testing.

Firefox support should not be announced until temporary loading, localhost fixture testing, popup behavior, Lab Session behavior, reports, and known limitations are verified and documented.

## Known Limitations

- LoginGuard is not a replacement for Chrome browser security warnings.
- LoginGuard is not a full penetration testing tool.
- LoginGuard does not prove that a site is secure.
- Header checks depend on browser-observed response headers and extension API behavior.
- Local development contexts such as `localhost` may intentionally omit production security headers.
- Lab Mode currently focuses on safe metadata and planner workflows.
- Planner-only checks describe future controlled lab checks but do not execute them.
- Firefox/WebExtensions compatibility is groundwork only and not a supported release target yet.
